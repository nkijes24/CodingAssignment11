Phase 2 Reflection

The loot array serves as the single source of truth because every loot item is stored there, and the screen is rebuilt from that array whenever the state changes.

I added quantity to the data model so each object can represent the item name, its value, and how many of that item the party collected. Total loot is calculated as value multiplied by quantity.

The updateUI() function performs the total calculation, split calculation, loot list rendering, button enabling, and section visibility control in one place.

State controls the interface because empty loot or an invalid party size hides results and disables the Split Loot button. Valid state makes the correct sections visible and usable.

Conditional logic prevents invalid state by checking party size, loot name, loot value, and quantity before data is added to the array.
